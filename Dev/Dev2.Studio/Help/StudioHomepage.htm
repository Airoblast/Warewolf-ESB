Click Workflow on the Ribbon above or Ctrl+W. A workflow is the start and end point of everything you build. All your work will be contained inside a workflow. You can have workflows being executed from inside other workflows.
Drag and drop tools from the Toolbox window onto the workflow and connect them using the small connectors. You can bring in other workflows or services (see step 3) that you have already created, either from the toolbox: Resources or from the explorer window.
Services can execute things that are outside this system, like database stored procedures, APIs or workflows on other Warewolf Servers. You can click on File / Home menus to do this. Typically you pass variables into a service and it passes back some other information when complete. This is done in the Input / Output (Mapping) section on the service or workflow.
All Variables hold any type of data you want and are simply used by typing [[variablename]]. Warewolf will maintain these in the Variables window for easy reference. Selecting Input / Output checkbox on the Variable list will determine what is required and what is returned when executing. You can define a recordset as well by going [[Recordsetname().Field]]
On the Ribbon above click Debug (F5) then Execute (F5 again). The Output window will show you how your application is executing and what it’s doing. You can also click View in Browser which will show you the URL and result of what your workflow will output if you connect to it externally.
