Warewolf Language

Explains how we use and refer to data in the system.

We use the Warewolf language to reference data in the Variable List. At run time data will be held in the variable list but will be needed in services etc. to actually get the work done. The Warewolf Language allows us to bridge that gap and utilise a specific piece of information as and where it’s needed.

There are two kinds of information: Scalar and Recordset. Scalar refers to an individual piece of information while Recordset refers to repeated information, much like a traditional recordset.

The notation for a Scalar variable is [[VariableName]].

The notation for a Recordset variable is [[Recordset(n).Field]].

n refers to the index of the recordset

See Language Rules for more information on notation.

Recordset or repeating variables can have a number of different uses, namely:

[[Recordset()]] - no Field

As Input: Only ever used when a tool needs to know a Recordset Name to perform work on, like Count Records Tool or For Each Tool.

As Result: Never

[[Recordset(n).Field]]

As Input: Behaves exactly like a scalar. This notation is used when getting information out of a recordset at a particular index. If n = 3, then the value of the field in row 3 will be used.

As Result: Behaves exactly like a scalar. The value of the variable will be overwritten with the new output value.

[[Recordset().Field]] - no Index

As Input: Assumes the last index and inserts the value at that index. This is so you dont have to track the last index.

As Result: Behaves exactly like a scalar. Appends one new record to the recordset from the last index.

[[Recordset(*).Field]]

As Input: Will execute each and every index in the recordset.

As Result: For every execution, the result will be written to the recordset starting at index 1 untill there are no more results. If indexes already exist then data at that index will be overwritten.

In all cases Indexing starts at 1 so when both input and output are using recordsets, then the same index is processed on Input as Output. In this way, where the index does not exist, know data will remain as is and only new records will be inserted with new data. Also when multiple recordset fields are specified in the output, they will all process with the same index so specifying multiple fields from the same recordset without specifying an index will put each of the outputs records into a new record with the same index.

Helpful Hints for the Recordset Language:

Tools:
Tools are designed to make life easy. Where you indicate * in a recordset for Input, every item will be processed.

Services:
DB, Webservice and Plugins endpoints are outside the control of Warewolf. If you enter a * we will cram all the info into that input field. If you want to shape the data first, use the JSON tool or XML. You can put multiple * recordset fields into the same input field by simply typing them in: “[[Family(*).LName]], [[Family(*).FName]] – [[Family(*).Nickname]] | “ in a single field will give you: “Simpson, Bartholomew – Bart | Simpson, Marjorie – Marge | …” etc.
